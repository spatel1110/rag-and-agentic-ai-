2. RAG Fundamentals 📚

Basics se shuru karte hain — RAG kya hai, components, chunking, embeddings aur vector stores.

2.1 RAG Kya Hai? 🤔

Bhai, RAG (Retrieval-Augmented Generation) basically ek smart library hai jahan se LLM apna jawab dhundhta hai — apne dimaag se hallucinate nahi karta. Pehle relevant documents retrieve karta hai, phir unko query ke saath LLM ko deta hai, taaki jawab factual aur up-to-date rahe.

2.3 Chunking Strategies ✂️

Bade documents ko chote pieces (chunks) mein todte hain. Chunk chota = precise match, bada = zyada context. Balance zaruri hai!

# Yahan hum document ko chunks mein tod rahe hain
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=512,        # ek chunk mein kitne characters
    chunk_overlap=80,      # context preserve karne ke liye overlap (10-20%)
    separators=["\n\n", "\n", ". ", " ", ""]  # hierarchical splitting
)
chunks = splitter.split_documents(docs)

3. Advanced RAG ⚡

Retrieval ko next level pe le jaate hain — better recall, better precision.

3.1 Retrieval Techniques

Sabse basic — query ka vector banao, aur vector DB se sabse paas wale k chunks le aao (cosine similarity).

results = db.similarity_search("RAG benefits", k=4)
# scores ke saath
results = db.similarity_search_with_score("RAG benefits", k=4)

MMR (Maximal Marginal Relevance) — relevance + diversity ka balance. Lambda 1 = pure relevance, 0 = pure diversity. Duplicate chunks avoid karta hai.

retriever = db.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 4, "fetch_k": 20, "lambda_mult": 0.5})

Hybrid Search = Dense (semantic) + Sparse (keyword/BM25). Jab exact keywords matter karte hain tab pure semantic se better hota hai.

Contextual Compression — retrieve karne ke baad noise hata ke sirf relevant part rakhta hai. LLM ko clean context milta hai.

from langchain.retrievers import ContextualCompressionRetriever

4.4 Tool Use & Function Calling 🛠️

Agent decide karta hai konsa tool kab call karna hai. Tools = Web search, Code executor, RAG retriever, Calculator, API caller.

# Yahan hum ek tool define kar rahe hain
from langchain_core.tools import tool

@tool
def search_docs(query: str) -> str:
    """Knowledge base se relevant info retrieve karta hai."""
    docs = retriever.invoke(query)
    return "\n".join(d.page_content for d in docs)

# Agent ko tools bind karo
llm_with_tools = llm.bind_tools([search_docs])

5.3 LangGraph Code Example — ReAct Agent

# Yahan hum ek simple ReAct agent bana rahe hain LangGraph se
from typing import Annotated, TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langgraph.prebuilt import ToolNode, tools_condition

# 1. State define karo (shared memory)
class AgentState(TypedDict):
    messages: Annotated[list, add_messages]

llm_with_tools = llm.bind_tools([search_docs])

# 2. Node function — agent ka brain
def agent_node(state: AgentState):
    return {"messages": [llm_with_tools.invoke(state["messages"])]}

# 3. Graph banao
graph = StateGraph(AgentState)
graph.add_node("agent", agent_node)
graph.add_node("tools", ToolNode([search_docs]))

# 4. Edges + conditional routing
graph.add_edge(START, "agent")
graph.add_conditional_edges("agent", tools_condition)  # tool chahiye ya nahi
graph.add_edge("tools", "agent")   # loop back
graph.add_edge("agent", END)

# 5. Compile aur invoke
app = graph.compile()
result = app.invoke({"messages": [("user", "RAG fusion explain karo")]})

7. Quick Cheatsheet 📝

Last-minute revision ke liye. Print/PDF bhi kar sakta hai.

RAG Cheatsheet

  • • Pipeline: Load → Split → Embed → Store → Retrieve → Generate
  • • Chunk size: Factoid 128–256 • General 256–512 • Complex 512–1024
  • • Overlap: 10–20% of chunk size
  • • Distance: Cosine (most common), Euclidean, Dot product
  • • MMR: lambda 1 = relevance, 0 = diversity
  • • Hybrid: BM25 (sparse) + embeddings (dense) via EnsembleRetriever
  • • HyDE: hypothetical answer embed karke search
  • • CRAG: grade docs → web fallback if irrelevant
  • • RAG Fusion: multi-query + Reciprocal Rank Fusion
db = Chroma.from_documents(chunks, emb)
r = db.as_retriever(search_type="mmr",
    search_kwargs={"k": 4})

Agentic AI Cheatsheet

  • • Agent = LLM + Memory + Tools + Planning + Loop
  • • ReAct: Reason → Act → Observe → repeat
  • • Plan & Execute: plan steps → execute → replan
  • • LangGraph: nodes=functions, edges=decisions, state=memory
  • • Conditional edges: routing based on state
  • • Memory: short-term, long-term, episodic, semantic
  • • Supervisor: ek boss agent → specialized sub-agents
  • • HITL: human-in-the-loop for risky/critical actions
  • • Eval: RAGAS — faithfulness, relevance, context precision/recall
graph = StateGraph(AgentState)
graph.add_conditional_edges(
    "agent", tools_condition)
